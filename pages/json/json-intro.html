1. ¿Qué es JSON?

Creado por Erick Escamilla C.

JSON (JavaScript Object Notation) es un formato ligero de intercambio de datos. Es fácil de leer y escribir para los humanos. Es fácil de analizar y generar para las máquinas. Se basa en un subconjunto del estándar del lenguaje de programación JavaScript ECMA-262 3ª edición - diciembre de 1999. JSON es un formato de texto completamente independiente del lenguaje, pero utiliza convenciones que resultan familiares a los programadores de la familia de lenguajes C, como C, C++, C#, Java, JavaScript, Perl, Python y muchos otros. Estas propiedades hacen de JSON un lenguaje de intercambio de datos ideal.

JavaScript Object Notation (JSON) es un formato basado en texto estándar para representar datos estructurados en la sintaxis de objetos de JavaScript. Es comúnmente utilizado para transmitir datos en aplicaciones web (por ejemplo: enviar algunos datos desde el servidor al cliente, así estos datos pueden ser mostrados en páginas web, o vice versa).

¿Qué es realmente JSON?

JSON es un formato de datos basado en texto que sigue la sintaxis de objeto de JavaScript, popularizado por Douglas Crockford. Aunque es muy parecido a la sintaxis de objeto literal de JavaScript, puede ser utilizado independientemente de JavaScript, y muchos entornos de programación poseen la capacidad de leer (convertir; parsear) y generar JSON.

Los JSON son cadenas - útiles cuando se quiere transmitir datos a través de una red. Debe ser convertido a un objeto nativo de JavaScript cuando se requiera acceder a sus datos. Ésto no es un problema, dado que JavaScript posee un objeto global JSON que tiene los métodos disponibles para convertir entre ellos.

Nota: Convertir una cadena a un objeto nativo se denomina parsing, mientras que convertir un objeto nativo a una cadena para que pueda ser transferido a través de la red se denomina stringification.

Un objeto JSON puede ser almacenado en su propio archivo, que es básicamente sólo un archivo de texto con una extension .json, y un MIME type (en-US) de application/json.

2. ¿Cuál es la estructura de JSON?

Creado por Erick Escamilla C.

Estructura de JSON.

JSON se basa en dos estructuras:

  • Una colección de pares nombre/valor. En varios lenguajes, esto se realiza como un objeto, un registro, una estructura, un diccionario, una tabla hash, una lista con claves o una matriz asociativa.

  • Una lista ordenada de valores. En la mayoría de los lenguajes, se realiza como una matriz, un vector, una lista o una secuencia.

Son estructuras de datos universales. Prácticamente todos los lenguajes de programación modernos las soportan de una forma u otra. Es lógico que un formato de datos que sea intercambiable con los lenguajes de programación se base también en estas estructuras.

En JSON, adoptan estas formas:

Un objeto es un conjunto desordenado de pares nombre/valor. Un objeto comienza con una llave izquierda { y termina con una llave derecha }. Cada nombre va seguido de : (dos puntos) y los pares nombre/valor están separados por una , (coma).

Un array es una colección ordenada de valores. Un array comienza con un corchete izquierdo [ y termina con un corchete derecho ]. Los valores se separan con una coma.

Un valor puede ser una cadena entre comillas dobles, un número, true, false, null, un objeto o un array. Estas estructuras pueden estar anidadas.

3. ¿Por qué usar JSON?

Creado por Erick Escamilla C.

JSON es un enfoque simple de serialización de objetos basado en la sintaxis de inicializadores de objetos de JavaScript. El código del inicializador (literal de objeto) se pone en una cadena y luego se interpreta utilizando el parser JSON. 

                    serializedObj='{firstName:"john", lastName:"doe"}';
                    ...
                    var person = JSON.parse(serializedObj);
                    document.getElementById("firstName").innerHTML=person.firstName;

JSON se utiliza ampliamente en varios frameworks y kits de herramientas AJAX para proporcionar una fácil serialización de objetos para las llamadas remotas. Hay una creciente comprensión de que tal vez JSON debe ser considerado como una opción al implementar SOA, por ejemplo, Dion Hinchcliffe publicó recientemente una entrada de blog sugiriendo que JSON (y otras tecnologías Web 2.0) debe ser considerado seriamente por los arquitectos de SOA (Service Oriented Architecture).

A continuación, se listan algunas de las ventajas de utilizar JSON en lugar de XML:

  • Manera totalmente automatizada de de-serializar/serializar objetos de JavaScript, con mínima o ninguna codificación.
  • Muy buena compatibilidad con todos los navegadores.
  • Formato conciso gracias al enfoque basado en el par nombre/valor
  • Deserialización rápida de objetos en JavaScript
  • Compatible con muchos conjuntos de herramientas AJAX y bibliotecas JavaScript
  • API sencilla (disponible para JavaScript y otros lenguajes)

4. Funciones (métodos) de JSON (Objeto global de JS)

Creado por Erick Escamilla C.

El objeto JSON contiene métodos para analizar la Notación de Objetos de JavaScript (JSON) y convertir valores a JSON. No se puede llamar ni construir, y aparte de sus dos propiedades de método, no tiene ninguna funcionalidad interesante propia.

Métodos estáticos

JSON.parse(text[, reviver])

Analiza la cadena de texto como JSON, opcionalmente transforma el valor producido y sus propiedades, y devuelve el valor. Cualquier violación de la sintaxis de JSON, incluyendo las diferencias entre JavaScript y JSON, provoca un SyntaxError. La opción reviver permite interpretar lo que el reponedor ha utilizado para sustituir a otros tipos de datos.

JSON.stringify(value[, replacer[, space]])

Devuelve una cadena JSON correspondiente al valor especificado, incluyendo opcionalmente sólo ciertas propiedades o sustituyendo los valores de las propiedades de una manera definida por el usuario. Por defecto, todas las instancias de undefined son reemplazadas por null, y otros tipos de datos nativos no soportados son censurados. La opción de reemplazo permite especificar otro comportamiento.

5. Diferencia entre JSON y JS

JSON vs. JS marzo 22, 2022

5. Diferencia entre JSON y JS

Creado por Erick Escamilla C.

Diferencia entre JSON y JS

Aunque la sintaxis de JSON es similar a la de los objetos de JavaScript, JSON es diferente de los objetos de JavaScript.

Diferencia entre JSON y Javascript
JSON Javascript Object
La clave del par clave/valor debe ir entre comillas dobles. La clave del par clave/valor puede ir sin comillas dobles.
JSON no puede contener funciones. Los objetos de JavaScript pueden contener funciones.
JSON puede ser creado y utilizado por otros lenguajes de programación. Los objetos JavaScript sólo pueden utilizarse en JavaScript.

JSON es una sintaxis para serializar objetos, matrices, números, cadenas, booleanos y nulos. Se basa en la sintaxis de JavaScript, pero es distinta de ella: un poco de JavaScript no es JSON.

  • Objetos y matrices

    Los nombres de las propiedades deben estar entre comillas dobles; las comas al final están prohibidas.

  • Números

    Los ceros a la izquierda están prohibidos. Un punto decimal debe ir seguido de al menos un dígito. No se admite NaN ni Infinito.

  • Cualquier texto JSON es una expresión JavaScript válida...

    ...Pero sólo en los motores de JavaScript que han implementado la propuesta para que todo el texto JSON sea válido ECMA-262. En los motores que no han implementado la propuesta, U+2028 LINE SEPARATOR y U+2029 PARAGRAPH SEPARATOR están permitidos en los literales de cadena y las claves de propiedades en JSON; pero su uso en estas características en los literales de cadena de JavaScript es un SyntaxError.

Considere este ejemplo donde JSON.parse() analiza la cadena como JSON y eval ejecuta la cadena como JavaScript: 

                    let code = '"\u2028\u2029"'
                    JSON.parse(code)  // evaluates to "\u2028\u2029" in all engines
                    eval(code)        // throws a SyntaxError in old engines

Otras diferencias son que sólo permite cadenas con comillas dobles y que no tiene disposiciones para indefinidos o comentarios. Para aquellos que deseen utilizar un formato de configuración más amigable basado en JSON, existe JSON5, utilizado por el compilador Babel, y el más comúnmente utilizado YAML

6. Ejemplo de JSON y PHP.

Creado por Erick Escamilla C.

Ejemplo de JSON y PHP

Un uso común de JSON es leer datos de un servidor web, y mostrar los datos en una página web.

Ejemplo 1
Archivo PHP:

PHP tiene algunas funciones incorporadas para manejar JSON.

Los objetos en PHP se pueden convertir en JSON utilizando la función de PHP json_encode(): 

                    <?php
                    $myObj->name = "John";
                    $myObj->age = 30;
                    $myObj->city = "New York";
                    
                    $myJSON = json_encode($myObj);
                    
                    echo $myJSON;
                    ?>
El cliente JavaScript

Aquí hay un JavaScript en el cliente, utilizando una llamada AJAX para solicitar el archivo PHP del ejemplo anterior: 

                    const xmlhttp = new XMLHttpRequest();
                    xmlhttp.onload = function() {
                      const myObj = JSON.parse(this.responseText);
                      document.getElementById("demo").innerHTML = myObj.name;
                    }
                    xmlhttp.open("GET", "demo_file.php");
                    xmlhttp.send();
Ejemplo 2: Array PHP
Archivo PHP:

Las matrices en PHP también se convertirán en JSON cuando se utilice la función de PHP json_encode(): 

                      <?php
                        $myArr = array("John", "Mary", "Peter", "Sally");

                        $myJSON = json_encode($myArr);

                        echo $myJSON;
                      ?>
Cliente Javascript

Aquí hay un JavaScript en el cliente, usando una llamada AJAX para solicitar el archivo PHP del ejemplo del array anterior: 

                      var xmlhttp = new XMLHttpRequest();
                      xmlhttp.onload = function() {
                        const myObj = JSON.parse(this.responseText);
                        document.getElementById("demo").innerHTML = myObj[2];
                      }
                      xmlhttp.open("GET", "demo_file_array.php", true);
                      xmlhttp.send();
