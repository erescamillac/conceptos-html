¿Qué es XAMPP?

Creado por Erick Escamilla C.

XAMPP es el entorno más popular de desarrollo con PHP.

XAMPP es una distribución de Apache completamente gratuita y fácil de instalar que contiene MariaDB, PHP y Perl. El paquete de instalación de XAMPP ha sido diseñado para ser increíblemente fácil de instalar y usar.

XAMPP es uno de los servidores web multiplataforma más utilizados, que ayuda a los desarrolladores a crear y probar sus programas en un servidor web local. Fue desarrollado por la organización Apache Friends, y su código fuente nativo puede ser revisado o modificado por el público. Está compuesto por el servidor HTTP Apache, MariaDB y el intérprete de diferentes lenguajes de programación como PHP y Perl. Está disponible en 11 idiomas y es compatible con diferentes plataformas como el paquete IA-32 de Windows y el paquete x64 de macOS y Linux.

XAMPP es una abreviatura en la que la X significa "Cross-Platform" (plataforma cruzada), la A significa "Apache", la M significa "MYSQL" y la Ps significan "PHP" y "Perl", respectivamente. Es un paquete de soluciones web de código abierto que incluye la distribución de Apache para muchos servidores y ejecutables de línea de comandos junto con módulos como el servidor Apache, MariaDB, PHP y Perl.

XAMPP ayuda a un host o servidor local a probar su sitio web y sus clientes a través de ordenadores y portátiles antes de lanzarlo al servidor principal. Es una plataforma que proporciona un entorno adecuado para probar y verificar el funcionamiento de proyectos basados en Apache, Perl, base de datos MySQL y PHP a través del sistema del propio host. Entre estas tecnologías, Perl es un lenguaje de programación utilizado para el desarrollo web, PHP es un lenguaje de scripting backend, y MariaDB es la base de datos más utilizada desarrollada por MySQL.

¿Para qué se usa (XAMPP)?

Creado por Erick Escamilla C.

El objetivo de XAMPP es crear una distribución fácil de instalar para desarrolladores que se están iniciando en el mundo de Apache. XAMPP viene configurado por defecto con todas los opciones activadas. XAMPP es gratuito tanto para usos comerciales como no comerciales.

Descripción completa del servidor web Apache. Apache HTTP Server es un software de servidor web gratuito y de código abierto para plataformas Unix con el cual se ejecutan el 46% de los sitios web de todo el mundo. ... Le permite a los propietarios de sitios web servir contenido en la web, de ahí el nombre de «servidor web»

XAMPP es una distribución de Apache que incluye varios software libres. El nombre es un acrónimo compuesto por las iniciales de los programas que lo constituyen: el servidor web Apache, los sistemas relacionales de administración de bases de datos MySQL y MariaDB, así como los lenguajes de programación Perl y PHP.

XAMPP es la abreviatura de cross-platform, Apache, MySQL, PHP y Perl, y te permite construir un sitio de WordPress sin conexión, en un servidor web local en tu ordenador. Esta solución sencilla y ligera funciona en Windows, Linux y Mac, de ahí lo de "multiplataforma".

Otros lenguajes para desarrollo web de BD en Internet

Creado por Erick Escamilla C.

PHP ha sido uno de los lenguajes de programación más antiguos y más utilizados para el desarrollo del lado del servidor, y está impulsando más de 42 millones de sitios hoy en día en la web en todo el mundo, de acuerdo con SimilarTech. PHP ha crecido mucho y ha madurado en el transcurso de 25 años, y la última versión importante, PHP7 trajo consigo muchas mejoras en el rendimiento, aumentando la velocidad hasta el 100% de la de PHP 5 y reduciendo el uso de memoria. También se dice que PHP Composer es uno de los mejores gestores de paquetes de terceros hoy en día, y el lenguaje posee ahora un puñado de potentes frameworks, incluyendo Laravel, Symfony, Zend y CodeIgniter. Pero las tecnologías siempre están cambiando, y algunas incluso han derrocado el reinado de PHP y han abierto las puertas a nuevas oportunidades por venir. Para aquellos que nunca se sintieron cómodos con PHP o que quieren hacerse con las nuevas tecnologías, aquí hay 10 alternativas para PHP que puedes utilizar para el desarrollo del lado del servidor:

Las mejores alternativas a PHP

  1. Javascript

    Node.js es actualmente la tecnología más caliente en el desarrollo web; es un entorno de ejecución para Javascript en el lado del servidor, en pocas palabras, permite utilizar Javascript para el desarrollo del lado del servidor. Ahora bien, esto no sería suficiente para hacerla tan popular como lo es, el verdadero estado del arte de esta tecnología es el hecho de que ejecuta código asíncrono en el servidor. Esto aumenta el rendimiento del servidor, haciéndolo más rápido y capaz de manejar más peticiones/conexiones.

    La comunidad de node está creciendo muy rápido, y el gestor de paquetes de node (NPM) cuenta con la mayor cantidad de paquetes, con más de 650.000 paquetes disponibles para que puedas construir tus aplicaciones.

  2. Java (EE)

    Java es el favorito de todos los tiempos para muchos desarrolladores, y ha reinado en la escena del desarrollo de software durante bastante tiempo. Java es bastante rápido por naturaleza, y dado que está fuertemente tipado, proporciona una mejor experiencia de desarrollo escribiendo código con menos errores. Muchos marcos de trabajo utilizan Java para el desarrollo del lado del servidor, como Spring y JEE.

  3. Python

    Lo creas o no, Python se utiliza ahora más que PHP en el desarrollo del lado del servidor, y por buenas razones también. El framework Django es actualmente muy famoso entre los principiantes y los profesionales. Python es un lenguaje bastante fácil, quizás el más fácil de esta lista, y Django está repleto de funcionalidades, siendo un framework con pilas incluidas que aporta mucha potencia a la mano del desarrollador. Además, muchas organizaciones prefieren utilizar Django, ya que aprovecha la seguridad y la escalabilidad. Sin olvidar a Flask, por supuesto, uno de los frameworks de Python más queridos, ya que es mínimo, fácil de coger y extremadamente flexible.

  4. Ruby

    Ruby es un potente lenguaje dinámico, que es extremadamente fácil de usar para los principiantes y se centra en la simplicidad y la productividad. Existe un debate continuo sobre si es más fácil que Python o no, pero digamos que son aproximadamente del mismo nivel. Rails, un marco de desarrollo web MVC de Ruby, llevó el lenguaje a nuevas cotas y se convirtió en el marco de referencia para muchos desarrolladores, ya que se centra en la creación rápida de prototipos y en el patrón de convención sobre configuración. Rails no es el único framework de Ruby que existe, están Sinatra, Cuba, Hanami y algunos otros.

  5. C#

    Los productos de Microsoft para el desarrollo web puede que no sean la opción más adecuada para muchos desarrolladores y tampoco son los más populares. Sin embargo, son una opción fuerte en términos de características y rendimiento. ASP.Net Core es la siguiente generación de su marco web ASP.Net más famoso. ASP.Net impulsa más de 17 millones de páginas en la web y es utilizado por Microsoft y Xfinity, entre otros. Si estás familiarizado con Java o C++, C# te resultará bastante intuitivo, y si ya estás familiarizado con C#, ASP.Net es definitivamente una opción para ti.

  6. Go

    Go es un rico lenguaje de programación compilado construido por Google, y se dice que es incluso más sencillo que Python. Es un lenguaje de programación concurrente y dado que también es compilado, esto contribuye a que sea un lenguaje rápido para el desarrollo web. El equipo que creó el lenguaje también hizo un increíble IDE que es un factor muy importante en el proceso de desarrollo.

  7. Erlang

    Erlang es una de las opciones más potentes de esta lista. Es un lenguaje de programación funcional concurrente que fue diseñado específicamente para manejar la cantidad masiva de tráfico en las aplicaciones en tiempo real. La fiabilidad y la disponibilidad en Erlang son de gran importancia, hasta el punto de que la adición de nuevas características podría hacerse sin problemas mientras la aplicación está manejando la cantidad masiva de transacciones. Aunque puede que no sea la mejor opción para la generación de HTML, sin duda es ideal para construir una API REST en su backend.

  8. Elixir

    Elixir es un lenguaje de programación funcional dinámico que utiliza la máquina virtual Erlang. Fue diseñado para la concurrencia, la escalabilidad y la simplicidad e inventado por un miembro del núcleo de Rails. Elixir posee un rico marco de trabajo llamado Phoenix, pero es relativamente joven en comparación con otros lenguajes y, por lo tanto, es posible que tengas que escribir cosas desde cero. Elixir y Phoenix también cuentan con flexibilidad y escalabilidad, los creadores también dijeron que fueron capaces de servir a 2 millones de usuarios simultáneamente, lo que abre la puerta a nuevas posibilidades que podrían no existir en otros frameworks.

  9. C++

    Puede que te sorprenda, pero no hay nada que C++ no pueda hacer, y eso incluye el desarrollo web. En realidad, hay numerosos frameworks web entre los que puedes elegir, y en términos de rendimiento y velocidad, podrían ser incluso más rápidos que los típicos frameworks. Sin embargo, ten en cuenta que el despliegue puede ser un poco problemático, pero ciertamente será divertido experimentar con ellos. Los frameworks más famosos son WT, su página oficial está hecha usando el framework y Crow, que es la versión de C++ de Flask, un mini framework web.

  10. Rust

    Ya que Rust es un lenguaje popular y muy querido, debemos mencionar que se pueden desarrollar aplicaciones web usando Rust. Aunque el lenguaje no es lo suficientemente maduro como para reemplazar frameworks como Rails, Django y Laravel, tiene las características básicas y en cuanto a servicio de API con una arquitectura del lado del cliente, podría hacer el trabajo. Dado que RUST es un lenguaje seguro, la mayoría de los peligros en el desarrollo web podrían ser eliminados sólo con el uso del lenguaje.

Breve historia de php

Creado por Erick Escamilla C.

PHP fue concebido en algún momento del otoño de 1994 por Rasmus Lerdorf. Las primeras versiones no publicadas se utilizaron en su página de inicio para llevar la cuenta de quién miraba su currículum en línea. La primera versión utilizada por otros estuvo disponible en algún momento a principios de 1995 y fue conocida como Personal Home Page Tools. Consistía en un motor de análisis sintáctico muy simplista que sólo entendía unas pocas macros especiales y una serie de utilidades de uso común en las páginas de inicio de entonces. Un libro de visitas, un contador y algunas otras cosas. El parser fue reescrito a mediados de 1995 y llamado PHP/FI Versión 2. El FI vino de otro paquete que Rasmus había escrito y que interpretaba datos de formularios html. Él combinó los scripts de las herramientas de la Página Personal con el Intérprete de Formularios y agregó soporte mSQL y así nació PHP/FI. PHP/FI creció a un ritmo asombroso y la gente comenzó a contribuir con el código.

Es difícil ofrecer estadísticas sólidas, pero se estima que a finales de 1996 PHP/FI se utilizaba en al menos 15.000 sitios web de todo el mundo. A mediados de 1997 este número había crecido a más de 50.000. A mediados de 1997 también se produjo un cambio en el desarrollo de PHP. Pasó de ser el proyecto favorito de Rasmus, al que habían contribuido un puñado de personas, a ser un esfuerzo de equipo mucho más organizado. El parser fue reescrito desde cero por Zeev Suraski y Andi Gutmans y este nuevo parser formó la base de la versión 3 de PHP. Mucho del código de utilidad de PHP/FI fue portado a PHP3 y mucho fue reescrito completamente.

Hoy en día (a finales de 1999), PHP/FI o PHP3 viene con un número de productos comerciales como el servidor web StrongHold de C2 y RedHat Linux. Una estimación conservadora basada en una extrapolación de las cifras proporcionadas por NetCraft (véase también Netcraft Web Server Survey) sería que PHP se utiliza en más de 1.000.000 de sitios en todo el mundo. Para ponerlo en perspectiva, eso es más sitios que los que ejecutan el servidor insignia de Netscape Enterprise en Internet.

Además, en el momento de escribir este artículo, se está trabajando en la próxima generación de PHP, que utilizará el potente motor de scripting Zend para ofrecer un mayor rendimiento, y también soportará la ejecución en servidores web distintos de Apache como módulo de servidor nativo.

Diferencia entre SQL y MySQL

Creado por Erick Escamilla C.

¿Qué es SQL?

SQL es un lenguaje que se usa para operar su base de datos. SQL es el lenguaje básico utilizado para todas las bases de datos. Hay cambios de sintaxis menores entre diferentes bases de datos, pero la sintaxis SQL básica sigue siendo en gran medida la misma. SQL es una abreviatura de Structured Query Language. De acuerdo con ANSI (American National Standards Institute), SQL es el lenguaje estándar para operar un sistema de administración de bases de datos relacionales.

SQL se usa para acceder, actualizar y manipular datos en una base de datos. Su diseño permite la gestión de datos en un RDBMS, como MYSQL. El lenguaje SQL también se usa para controlar el acceso a datos y para la creación y modificación de esquemas de Base de datos.

¿Qué es MYSQL?

Desarrollado a mediados de los años 90, MySQL fue una de las primeros gestores de bases de datos de código abierto disponibles en el mercado. Hoy en día existen muchas alternativas variantes de MySQL. Sin embargo, las diferencias entre las variantes no son significativas ya que usan la misma sintaxis, y la funcionalidad básica también permanece igual.

MySQL es un RDBMS que permite mantener organizados los datos que existen en una base de datos. MySQL se pronuncia como «My S-Q-L», pero también se llama «My Sequel.». Lleva el nombre de la hija del cofundador Michael Widenius. MySQL proporciona un acceso multiusuario a las bases de datos. Este sistema RDBMS se usa con la combinación de PHP y Apache Web Server, además de una distribución de Linux. MySQL usa el lenguaje SQL para consultar la base de datos.

Ahora veamos la diferencia entre SQL y MySQL

Diferencia entre SQL y MySQL
Parámetro SQL MySQL
Definición SQL es un lenguaje de consulta estructurado. Es útil para administrar bases de datos relacionales. MySQL es un RDBMS para almacenar, recuperar, modificar y administrar una base de datos utilizando SQL.
Complejidad Necesita aprender el lenguaje SQL para usarlo efectivamente. Está disponible a través de la descarga y la instalación.
Tipo SQL es un lenguaje de consulta. MySQL es un software de base de datos. Usó el lenguaje «SQL» para consultar la base de datos.
Soporte para conector SQL no proporciona conectores. MySQL ofrece una herramienta integrada llamada ‘MySQL workbench’ para diseñar y desarrollar bases de datos.
Propósito Para consultar y operar el sistema de base de datos. Permite el manejo, almacenamiento, modificación y eliminación de datos en formato tabular.
Uso El código SQL y los comandos se usan en varios sistemas DBMS y RDMS, incluido MYSQL. MYSQL se usa como una base de datos RDBMS.
