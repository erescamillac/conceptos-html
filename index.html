Modelo cliente-servidor

El modelo cliente-servidor es una estructura de aplicación distribuida que divide las tareas o la carga de trabajo entre los proveedores de un recurso o servicio, llamados servidores, y los solicitantes del servicio, llamados clientes. En la arquitectura cliente-servidor, cuando el ordenador cliente envía una solicitud de datos al servidor a través de Internet, el servidor acepta el proceso solicitado y entrega los paquetes de datos solicitados al cliente. Los clientes no comparten ninguno de sus recursos. Ejemplos del modelo cliente-servidor son el correo electrónico, la World Wide Web, etc.

¿Cómo funciona el modelo cliente-servidor?

En este artículo vamos a sumergirnos en el modelo Cliente-Servidor y echar un vistazo a cómo funciona Internet a través de los navegadores web. Este artículo nos ayudará a tener una base sólida de la WEB y nos ayudará a trabajar con las tecnologías WEB con facilidad.

  • Cliente: cuando hablamos de la palabra cliente, significa hablar de una persona o una organización que utiliza un servicio concreto. Del mismo modo, en el mundo digital, un cliente es un ordenador (host) capaz de recibir información o utilizar un servicio concreto de los proveedores de servicios (servidores).

  • Servidores: del mismo modo, cuando hablamos de la palabra Servidores, significa una persona o medio que sirve algo. En el mundo digital, un servidor es un ordenador remoto que proporciona información (datos) o acceso a determinados servicios.

Así que, básicamente, el cliente solicita algo y el servidor lo sirve siempre que el recurso (o servicio) requerido esté disponible.

HTTP

HTTP es un protocolo para la obtención de recursos, como los documentos HTML. Es la base de cualquier intercambio de datos en la Web y es un protocolo cliente-servidor, lo que significa que las solicitudes son iniciadas por el destinatario, normalmente el navegador web. Un documento completo se reconstruye a partir de los diferentes subdocumentos obtenidos, por ejemplo, el texto, la descripción del diseño, las imágenes, los vídeos, los scripts, etc.

Los clientes y los servidores se comunican mediante el intercambio de mensajes individuales (en lugar de un flujo de datos). Los mensajes enviados por el cliente, normalmente un navegador web, se denominan peticiones y los mensajes enviados por el servidor se denominan respuestas.

Diseñado a principios de los años 90, HTTP es un protocolo extensible que ha evolucionado con el tiempo. Es un protocolo de capa de aplicación que se envía a través de TCP, o de una conexión TCP cifrada con TLS, aunque en teoría podría utilizarse cualquier protocolo de transporte fiable. Debido a su extensibilidad, se utiliza no sólo para obtener documentos de hipertexto, sino también imágenes y vídeos o para enviar contenido a los servidores, como ocurre con los resultados de los formularios HTML. HTTP también puede utilizarse para obtener partes de documentos y actualizar las páginas web bajo demanda.

HTTPS

El protocolo de transferencia de hipertexto seguro (HTTPS) es la versión segura de HTTP, que es el principal protocolo utilizado para enviar datos entre un navegador web y un sitio web. HTTPS está encriptado para aumentar la seguridad de la transferencia de datos. Esto es especialmente importante cuando los usuarios transmiten datos sensibles, como al iniciar sesión en una cuenta bancaria, un servicio de correo electrónico o un proveedor de seguros médicos.

Cualquier sitio web, especialmente los que requieren credenciales de inicio de sesión, debe utilizar HTTPS. En los navegadores modernos, como Chrome, los sitios web que no utilizan HTTPS están marcados de forma diferente a los que sí lo hacen. Busque un candado verde en la barra de URL para indicar que la página web es segura. Los navegadores web se toman en serio el HTTPS; Google Chrome y otros navegadores marcan todos los sitios web que no utilizan HTTPS como no seguros.

URL

Junto con el Hipertexto y HTTP, las URL son uno de los conceptos claves de la Web. Es el mecanismo usado por los navegadores para obtener cualquier recurso publicado en la web.

URL significa Uniform Resource Locator (Localizador de Recursos Uniforme). Una URL no es más que una direccion que es dada a un recurso único en la Web. En teoria, cada URL valida apunta a un único recurso. Dichos recursos pueden ser páginas HTML, documentos CSS, imagenes, etc. En la práctica, hay algunas excepciones, siendo la más común una URL apuntando a un recurso que ya no existe o que ha sido movido. Como el recurso representado por la URL y la URL en si son manejadas por el servidor Web, depende del dueño del servidor web manejar ese recurso y su URL asociada adecuadamente.

http es el protocolo. La primera parte de la URL indica qué protocolo debe usar el navegador. Un protocolo es un método establecido para intercambiar o transferir datos alrededor de una red informática. Por lo general, para sitios web es el protocolo HTTP o su versión segura, HTTPS. La Web requiere uno de estos dos, pero los navegadores también saben cómo manejar otros protocolos como mailto: (para abrir un cliente de correo) o ftp: para manejar la transferencia de archivos, así que no se sorprenda si ve tales protocolos.

www.example.com es el nombre de dominio. Indica qué servidor web se solicita. Alternativamente, es posible usar directamente un dirección IP, pero debido a que es menos conveniente, no se usa con frecuencia en la Web.

:80 es el puerto. Indica la "puerta" técnica utilizada para acceder a los recursos en el servidor web. Por lo general, se omite si el servidor web utiliza los puertos estándar del protocolo HTTP (80 para HTTP y 443 para HTTPS) para otorgar acceso a sus recursos. De lo contrario es obligatorio.

/path/to/myfile.html es la ruta al recurso en el servidor web. En los primeros días de la Web, una ruta como esta representaba la ubicación de un archivo físico en el servidor web. Hoy en día, es principalmente una abstracción manejada por servidores web sin ninguna realidad física.

TCP/IP

TCP/IP son las siglas de Transmission Control Protocol/Internet Protocol (Protocolo de Control de Transmisión/Protocolo de Internet) y es un conjunto de protocolos de comunicación utilizados para interconectar dispositivos de red en Internet. TCP/IP también se utiliza como protocolo de comunicación en una red informática privada (una intranet o extranet).

Todo el conjunto de protocolos IP -un conjunto de reglas y procedimientos- se conoce comúnmente como TCP/IP. TCP e IP son los dos protocolos principales, aunque hay otros incluidos en el conjunto. El conjunto de protocolos TCP/IP funciona como una capa de abstracción entre las aplicaciones de Internet y el tejido de enrutamiento y conmutación.

TCP/IP especifica cómo se intercambian los datos en Internet proporcionando comunicaciones de extremo a extremo que identifican cómo deben dividirse en paquetes, dirigirse, transmitirse, enrutarse y recibirse en el destino. TCP/IP requiere poca gestión centralizada y está diseñado para que las redes sean fiables, con la capacidad de recuperarse automáticamente del fallo de cualquier dispositivo de la red.

FTP

El FTP (Protocolo de Transferencia de Archivos) se utiliza para comunicar y transferir archivos entre ordenadores en una red TCP/IP (Protocolo de Control de Transmisión/Protocolo de Internet), también conocida como Internet. Los usuarios, a los que se les ha concedido acceso, pueden recibir y transferir archivos en el servidor del Protocolo de Transferencia de Archivos (también conocido como host/sitio FTP).

¿Cómo funciona el FTP?

La conexión FTP necesita que dos partes se establezcan y se comuniquen en la red. Para ello, los usuarios necesitan tener permiso proporcionando credenciales al servidor FTP. Algunos servidores FTP públicos pueden no requerir credenciales para acceder a sus archivos. Esta práctica es común en el llamado FTP anónimo.

Hay dos canales de comunicación distintos mientras se establece una conexión FTP. El primero se llama canal de comandos, donde se inicia la instrucción y la respuesta. El otro se llama canal de datos, donde se produce la distribución de datos.

Para obtener o transferir un archivo, un usuario autorizado utilizará el protocolo para solicitar la creación de cambios en el servidor. A cambio, el servidor concederá ese acceso. Esta sesión se conoce como el modo de conexión activa.

La distribución en modo activo puede tener un problema si un cortafuegos protege la máquina del usuario. El cortafuegos no suele permitir ninguna sesión no autorizada de una parte externa.

El modo pasivo se utiliza si se produce ese problema. En este modo, el usuario establece tanto el canal de comandos como el de datos. Este modo pide al servidor que escuche en lugar de intentar crear una conexión con el usuario.

2. ¿Qué es, y cuáles son sus elementos/componentes?

Hipervínculo

Un hipervínculo es un elemento de un documento HTML que enlaza con otra parte del documento o con otro documento. En las páginas web, los hipervínculos suelen ser de color púrpura o azul y a veces están subrayados.

Elemento HTML Anchor <a>

El Elemento HTML Anchor <a> crea un enlace a otras páginas de internet, archivos o ubicaciones dentro de la misma página, direcciones de correo, o cualquier otra URL.

Páginas Web

Documento que se puede visualizar en un navegador web como Firefox, Google Chrome, Opera, Microsoft Internet Explorer o Edge, o Safari de Apple. También suelen llamarse simplemente "páginas".

Una página web es un documento, normalmente escrito en HTML, que se visualiza en un navegador de Internet. Se puede acceder a una página web introduciendo una dirección URL en la barra de direcciones del navegador. Una página web puede contener texto, gráficos e hipervínculos a otras páginas web y archivos.

Una página web se utiliza a menudo para proporcionar información a los espectadores, incluyendo imágenes o vídeos para ayudar a ilustrar temas importantes. Una página web también puede utilizarse como método para vender productos o servicios a los espectadores. Un sitio web se compone de varias páginas, como nuestro sitio web Computer Hope.

Cuando usted hace clic en un enlace proporcionado por un motor de búsqueda, está accediendo a una página web. Internet consta de millones de páginas web, y cada día se añaden más.

¿Cuáles son los elementos de una página web?

Cada página web es diferente. Sin embargo, la mayoría de las páginas contienen algunos o todos los elementos que figuran en esta página web. A continuación se ofrece un resumen de los principales elementos de una página web.

  1. El nombre del sitio web, el blog, el logotipo o el nombre de la empresa suelen aparecer en la esquina superior izquierda de cada página web. El logotipo también puede incluir un eslogan o una breve declaración sobre el sitio para dar a los visitantes una comprensión instantánea del mismo. Además, esta sección de la página web suele ser un enlace que puede remitir a la página de inicio.
  2. La búsqueda permite al visitante buscar en un sitio web información relacionada y debe estar disponible en todas las páginas.
  3. La barra de navegación, o menú de una página web, suele encontrarse en la parte superior o izquierda de cada página web. Debe incluir enlaces a cada una de las secciones principales del sitio web.
  4. Los banners publicitarios pueden mostrarse en diferentes lugares de una página web y ayudan a sufragar los gastos de funcionamiento de un sitio web y una empresa. Los banners publicitarios suelen encontrarse en la parte superior, izquierda, derecha o inferior de una página y también pueden incluirse en el contenido.
  5. Los enlaces para compartir en redes sociales permiten a los visitantes compartir su sitio con otros compañeros de trabajo, amigos y familiares en las redes sociales.
  6. Las migas de pan ayudan al usuario a saber en qué parte del sitio web se encuentra. También proporcionan una forma de que el usuario vuelva a otras secciones principales del sitio web o las explore.
  7. El título debe incluirse en la parte superior de cada página web y debe crearse utilizando la etiqueta HTML <h1>.
  8. El párrafo de apertura es uno de los más importantes de la página web y debe ayudar a atraer al visitante a la lectura de la página web. Una forma de atraer a los visitantes es incluir una imagen principal cerca del párrafo inicial.
  9. Cada página web debe estar dividida en otros títulos que permitan al visitante hojear y encontrar fácilmente lo más interesante de la página. Al crear una página web, esto se puede hacer utilizando las etiquetas HTML <h2> a <h6>.
  10. Disponer de un método que permita al visitante dar su opinión es también un requisito para la mayoría de las páginas web. Ofrecer al visitante enlaces o botones para que haga clic en su opinión es un buen paso para saber si una página web es útil o no.
  11. En la mayoría de las páginas web, la información y las herramientas adicionales, como un botón para imprimir la página, también pueden ser útiles para los usuarios.
  12. El pie de página debe incluir cualquier otra información importante de la empresa o del sitio web y debe incluirse en todas las páginas web para ayudar al visitante a seguir utilizando el sitio.
  13. Los derechos de autor y cualquier aviso legal o de privacidad también deberían figurar en todas las páginas web. Esto no sólo puede enlazar con la información legal pertinente, sino que es una indicación para la mayoría de los visitantes de que han llegado al final de la página web.
  14. Por último, un botón o enlace de "vuelta al principio" en la parte inferior de la página ayuda al visitante a volver a los enlaces del menú u otras herramientas.

Sitios Web

Un sitio web es un conjunto de páginas web interconectadas y de acceso público que comparten un mismo nombre de dominio. Los sitios web pueden ser creados y mantenidos por un individuo, un grupo, una empresa o una organización para servir a una variedad de propósitos.

Sistemas Web

El sistema de información web, o sistema de información basado en la web, es un sistema de información que utiliza las tecnologías web de Internet para ofrecer información y servicios, a los usuarios o a otros sistemas de información/aplicaciones. Se trata de un sistema de software cuyo objetivo principal es publicar y mantener datos utilizando principios basados en el hipertexto.

Arquitectura de los sistemas basados en la web

Arquitecturas cliente-servidor y web de 2 niveles

Normalmente, cuando se navega por Internet, se utiliza un navegador web como Internet Explorer o Mozilla Firefox. El ordenador que ejecuta un navegador se llama cliente, mientras que la máquina que proporciona las páginas web se llama servidor.

Cuando usted se conecta a un proveedor de servicios de Internet (ISP), por ejemplo, Blue Yonder o Aol, su ordenador establece una conexión de red con un servidor web. En esta situación, su ordenador es en efecto un cliente, que está conectado a un servidor web del ISP. El servidor web, como su nombre indica, sirve a su navegador páginas web (por ejemplo, páginas HTML, ASPX, JSP, etc.).

Este sencillo escenario, en el que el servidor web está conectado a uno o varios clientes, se conoce como modelo de arquitectura de 2 niveles.

Arquitecturas de 3 niveles

En general, las aplicaciones informáticas constan de tres tipos de funcionalidades distintas y diferenciadas.

  • Servicios de presentación: Se manifiestan en forma de visualización de información y facilidades para la introducción de datos por parte del usuario. Por lo general, es el front-end para la interacción con el usuario. Por ejemplo, el inicio de sesión requiere una interacción en forma de recogida de información sobre el nombre de usuario y la contraseña mediante un formulario HTML.
  • Lógica funcional: Toda aplicación incluye algún tipo de procesamiento de datos y esto también puede implicar la interactividad con la base de datos. Por ejemplo, la autenticación de usuarios requiere que la unidad lógica lea las combinaciones de nombre de usuario y contraseña de una base de datos y las compare hasta llegar a una buena comparación (con suerte).
  • Gestión de datos: Los datos, su almacenamiento, inserción y recuperación, su gestión y alteración son fundamentales para las aplicaciones informáticas. Por ejemplo, un sistema de gestión de bases de datos (DBMS) es necesario para la gestión de los nombres de usuario y las contraseñas asociadas, sus propietarios, etc.

Aplicaciones Web

Una aplicación web es un software o programa al que se puede acceder mediante cualquier navegador web. Su frontend suele crearse utilizando lenguajes como HTML, CSS, Javascript, que son soportados por los principales navegadores. Mientras que el backend puede utilizar cualquier pila de programación como LAMP, MEAN, etc. A diferencia de las aplicaciones móviles, no existe un SDK específico para desarrollar aplicaciones web.

Las aplicaciones web cobraron protagonismo con la llegada del movimiento del software como servicio (SaaS).

Ventajas de las aplicaciones Web

  1. En comparación con las aplicaciones de escritorio, las aplicaciones web son más fáciles de mantener, ya que utilizan el mismo código en toda la aplicación. No hay problemas de compatibilidad.
  2. Las aplicaciones web pueden utilizarse en cualquier plataforma: Windows, Linux, Mac... ya que todas son compatibles con los navegadores modernos.
  3. La aprobación de la tienda de aplicaciones móviles no es necesaria en las aplicaciones web.
  4. Liberado en cualquier momento y en cualquier forma. No es necesario recordar a los usuarios que actualicen sus aplicaciones.
  5. Puedes acceder a estas aplicaciones web las 24 horas del día y los 365 días del año desde cualquier PC.
  6. Puede utilizar el ordenador o su dispositivo móvil para acceder a los datos necesarios.
  7. Las aplicaciones web son una opción rentable para cualquier organización. Las licencias de asiento para el software de escritorio son caras, mientras que los SasS, por lo general, son de pago.
  8. Las aplicaciones basadas en la web son aplicaciones habilitadas para Internet a las que se accede a través del navegador web del móvil. Por lo tanto, no es necesario descargarlas ni instalarlas.

Aplicación web versus sitio web

A continuación se indican las principales diferencias entre una aplicación web y un sitio web:

Aplicación Web VS. Sitio web
Parámetro Aplicación web Sitio web
Creado para Una aplicación web está diseñada para interactuar con el usuario final Un sitio web se compone principalmente de contenido estático. Es de acceso público para todos los visitantes.
Interacción con el usuario En una aplicación web, el usuario no sólo lee el contenido de la página, sino que también manipula los datos restringidos. Un sitio web ofrece contenidos visuales y de texto que el usuario puede ver y leer, pero que no afectan a su funcionamiento.
Autenticación Las aplicaciones web necesitan autenticación, ya que ofrecen un abanico de opciones mucho más amplio que los sitios web. La autentificación no es obligatoria para los sitios web informativos. El usuario puede pedir registrarse para obtener una actualización periódica o para acceder a opciones adicionales. Estas funciones no están disponibles para los visitantes del sitio web no registrados.
Tarea y complejidad Las funciones de las aplicaciones web son bastante más altas y complejas en comparación con un sitio web. El sitio web muestra los datos y la información recogidos en una página específica.
